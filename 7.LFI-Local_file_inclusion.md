La vulnerabilità Local File Inclusion (LFI) è una vulnerabilità della sicurezza informatica che si verifica quando un'applicazione Web non convalida correttamente l'input dell'utente, consentendo a un utente malintenzionato di accedere ai file locali sul server Web.

In molti casi, gli aggressori sfruttano la vulnerabilità LFI abusando dei parametri di input nell'applicazione web. I parametri di input sono dati che gli utenti inseriscono nell'applicazione web, come URL o campi modulo. Gli aggressori possono manipolare i parametri di input per includere percorsi di file locali nella richiesta, il che potrebbe consentire loro di accedere ai file sul server web. Questa tecnica è nota come “Path Traversal” ed è comunemente utilizzata negli attacchi LFI.

Nell'attacco Path Traversal, l'aggressore utilizza caratteri speciali e caratteri di escape nei parametri di input per navigare nelle directory del server Web e accedere ai file in posizioni sensibili del sistema.

Ad esempio, l'aggressore potrebbe includere "../" nel parametro di input per navigare nella struttura delle directory e accedere ai file in posizioni sensibili del sistema.

Per prevenire attacchi LFI, è importante che gli sviluppatori di applicazioni web convalidino e filtrino adeguatamente l'input dell'utente, limitando l'accesso alle risorse di sistema e garantendo che i file possano essere inclusi solo da posizioni consentite. Inoltre, le aziende dovrebbero implementare misure di sicurezza adeguate, come la crittografia dei file e la limitazione dell’accesso non autorizzato alle risorse di sistema.
